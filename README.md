CS 360 Project Three Reflection
App Requirements and Goals

The goal of my mobile application was to design and develop a functional Android app that solves a real user need while applying mobile development best practices. My app, LetMeTellYouWhatToDo, was designed to help users organize tasks in a simple and structured way. The primary user need addressed by this app was task management and productivity. Users need a quick and intuitive way to create, view, and manage tasks without unnecessary complexity.

The requirements included building multiple screens, implementing navigation between activities, storing user input, applying UI design principles, and ensuring compatibility across Android versions. The app also needed to demonstrate secure and efficient coding practices.

Screens, Features, and User-Centered Design

To support user needs, the app included the following screens and features:

Login/Create Account screen

Task Grid display screen

Add/Edit Task functionality

Clean navigation between activities

RecyclerView for organized task display

The UI was designed with simplicity and clarity in mind. I focused on readable text, logical button placement, consistent color themes, and intuitive layout spacing. The goal was to reduce user confusion and make actions obvious. For example, important actions like adding a task were clearly visible and accessible.

The design was successful because it minimized clutter while maintaining functionality. Users can quickly understand how to navigate the app without instructions, which is a core principle of user-centered design.

Coding Approach and Development Strategy

When coding the app, I followed a structured and modular approach. I separated responsibilities between activities, adapters, and layouts to maintain readability and organization. I used Kotlin best practices, clear variable naming, and consistent formatting to keep the code maintainable.

I approached development step by step:

Built the UI layout first

Connected UI elements to Kotlin code

Implemented functionality incrementally

Tested each feature before moving forward

This incremental strategy prevented large debugging issues later. These same techniques can be applied in future projects, especially larger applications, by keeping code modular, readable, and scalable.

Testing and Debugging

To ensure the app was functional, I tested it using the Android emulator and physical device testing when possible. I verified:

Screen navigation worked correctly

Buttons triggered expected actions

RecyclerView displayed tasks properly

No crashes occurred during normal use

Testing is important because it ensures reliability and a positive user experience. During testing, I identified issues such as layout misalignment and activity transition errors. Fixing these issues improved the stability and polish of the final product.

Innovation and Overcoming Challenges

One challenge I encountered was managing screen transitions and properly connecting UI components to backend logic. I had to carefully debug intent handling and lifecycle behavior to prevent crashes.

I also innovated by improving the task display layout beyond the basic requirements, ensuring the app felt more like a real production application rather than a simple prototype. Adjusting layout spacing and improving visual organization enhanced the overall usability.

Strongest Demonstration of Skills

The component where I most successfully demonstrated my skills was the implementation of the RecyclerView and task management logic. This required understanding adapters, data handling, and UI updates. It showcased my ability to combine frontend design with backend logic effectively.

This project demonstrates my growth in Android development, problem solving, and applying user-centered design principles. It represents a strong addition to my professional portfolio and reflects my ability to build functional, structured, and user-focused mobile applications.
